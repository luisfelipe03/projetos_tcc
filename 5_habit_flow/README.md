# **Proposta do Projeto**

Desenvolver um aplicativo Flutter de média complexidade que funcione como um **rastreador de hábitos**, permitindo que os usuários cadastrem hábitos diários, marquem-nos como concluídos e acompanhem o progresso ao longo do tempo.

O foco do projeto está na **implementação do front-end**, utilizando protótipos de alta fidelidade como base para a construção da interface.

---

# **Objetivo**

O objetivo deste projeto é desenvolver **toda a interface do aplicativo** a partir de protótipos de alta fidelidade, avaliando a capacidade de modelos de IA em gerar código Flutter funcional, organizado e fiel ao design proposto.

### Funcionalidades previstas

* Cadastro, edição e remoção de hábitos
* Notificações de lembrete para hábitos diários
* Visualização de gráficos de progresso
* Login social com Google e Apple utilizando Firebase Authentication
* Persistência de dados com Cloud Firestore
* Registro diário do hábito com suporte a fotos
* Suporte a **tema claro (Light Mode) e escuro (Dark Mode)**

---

# **Métricas de Avaliação**

* **Funcionalidade**
  A aplicação deve permitir adicionar, visualizar, editar e remover hábitos de forma clara e intuitiva.

* **Qualidade do Código**
  O código deve ser organizado, seguir as boas práticas do Flutter e estar em conformidade com o analisador oficial da linguagem.

* **Fidelidade ao Design**
  A interface deve respeitar o visual definido nos protótipos de alta fidelidade disponíveis em:
  [Link para o protótipo](https://stitch.withgoogle.com/projects/10504349076465471902)

* **Esforço de Refatoração**
  O código gerado deve exigir o mínimo possível de ajustes manuais para atender aos critérios de funcionalidade, qualidade e fidelidade ao design.
