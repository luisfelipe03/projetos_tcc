# **Proposta do Projeto**

Desenvolver um aplicativo Flutter de média complexidade que funcione como o rastreador de hábitos, permitindo aos usuários adicionar os seus hábitos diários, marcar os hábitos como concluídos e visualizar o progresso ao longo do tempo. A interface deve ser intuitiva e responsiva, seguindo as melhores práticas de design do Flutter.

# **Métricas de Avaliação**

* **Funcionalidade:**
  A tela deve permitir adicionar, visualizar, editar e remover tarefas de forma direta e intuitiva.
* **Qualidade do Código:**
  O código precisa ser organizado, seguindo padrões do Flutter e passando pelas verificações do analisador oficial.
* **Fidelidade ao Design:**
  A interface precisa respeitar o visual já estabelecido no [FlutterBook](https://github.com/taburity/flutter_book_v3) original.
* **Tempo de Desenvolvimento:**
  O processo deve ser rápido, aproveitando ao máximo a automação proporcionada pelos prompts.
* **Esforço de refatoração:**
  O código gerado deve demandar o mínimo de ajustes manuais para atingir os objetivos acima.

## Resultados
Os resultados abotidos foram muito positivos, atendendo a todas as métricas estabelecidas. A tela de ToDo List foi implementada com sucesso, permitindo ao usuário adicionar, visualizar, editar e remover tarefas de maneira intuitiva. O código gerado seguiu as boas práticas do Flutter, passando sem problemas pelas verificações do analisador oficial, conforme evidenciado na imagem abaixo:
